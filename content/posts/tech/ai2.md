大语言模型（LLM）的本质是“文本概率的猜谜高手”，通过海量数据训练实现“智能涌现”。随后依次介绍了让模型记住对话历史的“上下文”（Context）、指导模型行为的“提示词工程”（Prompt）、解决幻觉问题的“检索增强生成”（RAG）、赋予模型行动能力的“智能体”（Agent），以及实现工具调用的“函数调用”（Function Calling）。统一交互标准的“模型上下文协议”（MCP）和按需加载工具的“技能”（Skill）

# LLM 
LLM (Large Language Model): 大语言模型，指通过海量文本数据训练，具有复杂逻辑处理和自然语言理解能力的深度学习模型。
- 幻觉
  大模型本质上是根据概率猜测下一个词，当面对它训练数据中未覆盖或不熟悉的领域时，它仍会尝试构建看似合理的回答，从而产生事实错误，这就是所谓的“幻觉”。

所有大语言模型（LLM）的本质都是“文本概率的猜谜高手”，它们通过海量文本数据训练，学会了预测下一个最可能的词，但本身不具备思考能力，也无法直接操作电脑或手机等外部环境。这种基于概率预测的机制，是理解后续所有概念的基础。工程师通过将参数规模从百万级提升到千亿级，并喂给它全世界的数据，最终触发了“智能涌现”现象，使其从早期的“人工智障”变得异常强大。明确这一根本原理，有助于开发者更理性地使用和期待大模型的能力

# 三个关键概念
## Context
“上下文”（Context），它如同大模型的“临时记忆小本本”，因为模型自身没有记忆，每次对话都是全新的，必须通过传递历史对话内容来维持连贯性。

## Prompt
“提示词工程”（Prompt），这是告诉大模型“做什么”和“怎么做”的说明书，提示词的质量直接决定了输出结果的优劣，例如通过设定角色（如“精通Go语言的专家”）可以精准控制输出格式。

## RAG
“检索增强生成” RAG (Retrieval-Augmented Generation)，旨在解决大模型的“幻觉”问题，通过构建外部知识库，让模型在回答前先检索相关资料，并将其作为上下文输入，从而确保回答基于真实可靠的信息，尤其适用于代码生成等需要准确知识的场景。

# Agent
- 普通大模型仅具备文本生成能力，通过问答与用户交互；而 Agent 在此基础上增加了自主行动能力，能够根据任务需求调用外部工具、执行代码或操控电脑环境，从而完成实际操作任务。

让大模型从“只会说”到“能做”。首先引入了“智能体”（Agent）的概念，它在大模型文本生成能力的基础上，增加了自主行动的能力，例如能够直接读写项目文件、操作开发环境，像Cursor、GitHub Copilot等工具本质上就是面向编程的Agent。

## “函数调用”（Function Calling）机制
Agent需要知道何时调用哪个工具，这便引出了“函数调用”（Function Calling）机制。其原理是通过提示词告知模型可用的工具列表及其参数，模型根据问题决定调用哪个工具并以特定格式输出指令，由Agent解析执行后再将结果返回给模型，从而实现了模型对外部工具的主动调用。

## MCP
MCP (Model Context Protocol): 模型上下文协议，由 Anthropic 推出，旨在统一大模型与外部工具及系统之间交互的规范，解决不同厂商接口不兼容的问题。

## Skill
早期的MCP模式需要将所有工具信息塞入提示词，消耗大量Token且挤占上下文空间。由此，“技能”（Skill）概念应运而生，
Skill: 指在开发中将工具能力标准化、模块化，通过“按需加载”而非一次性全部塞入系统提示词的方式，减少 Token 消耗并提升系统灵活性。
